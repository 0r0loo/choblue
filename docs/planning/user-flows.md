# 유저 플로우

---

## 1. 고객 플로우 (키오스크)

### 1-1. 대기 화면 → 터치하여 시작

**사용자가 보는 것:**
- 전체 화면에 매장 로고, 환영 메시지("주문하시려면 화면을 터치해 주세요"), 배경 이미지 또는 프로모션 슬라이드
- 하단에 매장 영업 정보 (선택적)

**가능한 동작:**
- 화면 아무 곳이나 터치 → 메뉴 화면으로 전환

**다음 단계:**
- 메뉴 화면 (1-2)

**엣지 케이스:**
- **영업 시간 외**: "현재 영업 시간이 아닙니다. 영업시간: 09:00~21:00" 메시지 표시. 터치해도 메뉴 화면으로 진입 불가
- **매장 임시 휴무**: "오늘은 휴무입니다." 메시지 표시
- **판매 가능한 상품이 0개**: "현재 주문 가능한 메뉴가 없습니다." 메시지 표시

---

### 1-2. 카테고리 선택 + 메뉴 탐색

**사용자가 보는 것:**
- **좌측 또는 상단**: 카테고리 탭 목록 (예: 커피, 논커피, 디저트, 음료)
- **중앙**: 선택된 카테고리의 상품 그리드 (상품 이미지, 이름, 가격)
- **우측 또는 하단**: 장바구니 미니 패널 (담긴 상품 수, 합계 금액)
- 품절 상품은 흑백 처리 + "품절" 뱃지, 터치 불가

**가능한 동작:**
- 카테고리 탭 터치 → 해당 카테고리 상품 목록으로 전환
- 상품 카드 터치 → 옵션 선택 모달 열기 (1-3)
- 장바구니 패널 터치 → 장바구니 상세 화면 (1-4)
- "처음으로" 버튼 → 장바구니 초기화 확인 후 대기 화면 복귀

**다음 단계:**
- 상품 터치 → 옵션 선택 모달 (1-3)
- 장바구니 터치 → 장바구니/결제 화면 (1-4)

**엣지 케이스:**
- **카테고리에 상품이 없는 경우**: 해당 카테고리는 탭에 표시하지 않음 (숨김 또는 비활성화 카테고리도 미표시)
- **5분간 터치 없음**: "계속 주문하시겠습니까?" 확인 모달 표시. 15초 내 터치 없으면 장바구니 초기화 후 대기 화면 복귀

---

### 1-3. 옵션 선택

**사용자가 보는 것:**
- 바텀 시트 또는 모달 형태
- 상품 이미지(크게), 상품명, 기본 가격, 상품 설명
- 옵션 그룹별 선택 영역:
  - **필수 옵션** (예: 사이즈 — 레귤러/라지): 라디오 버튼, 기본값 선택됨, 추가 금액 표시 (+500원)
  - **선택 옵션** (예: 토핑 추가 — 샷 추가, 시럽 변경): 체크박스, 복수 선택 가능, 추가 금액 표시
- 수량 조절 (-, 수량, +), 최소 1개
- 합계 금액 (기본가 + 옵션 추가금 x 수량) 실시간 반영
- "장바구니에 담기" 버튼 (합계 금액 표시)
- "취소" 또는 X 버튼

**가능한 동작:**
- 옵션 선택/해제 → 합계 금액 즉시 업데이트
- 수량 변경 → 합계 금액 즉시 업데이트
- "장바구니에 담기" → 장바구니에 추가, 모달 닫힘, 메뉴 화면으로 복귀
- "취소" / X → 모달 닫힘, 아무것도 추가 안 됨

**다음 단계:**
- 장바구니 담기 → 메뉴 화면 (1-2)으로 복귀 (장바구니 패널에 수량 업데이트)
- 취소 → 메뉴 화면 (1-2)으로 복귀

**엣지 케이스:**
- **필수 옵션 미선택**: 기본값이 항상 선택되어 있으므로 발생하지 않음. 기본값 없는 필수 옵션의 경우 "장바구니에 담기" 버튼 비활성화 + "사이즈를 선택해 주세요" 안내
- **옵션이 없는 상품**: 옵션 영역 없이 수량만 표시
- **수량 0 이하**: 최소 1개로 제한, 마이너스 버튼 비활성화

---

### 1-4. 장바구니 확인

**사용자가 보는 것:**
- 장바구니에 담긴 상품 목록: 상품명, 선택한 옵션, 수량, 소계
- 각 항목별: 수량 변경 (-, +), 삭제 (X) 버튼
- 총 주문 금액 (모든 항목 소계의 합)
- "결제하기" 버튼 (총 금액 표시)
- "더 담기" 버튼 → 메뉴 화면으로 복귀
- "전체 삭제" 버튼

**가능한 동작:**
- 수량 변경 → 소계, 총 금액 즉시 업데이트
- 삭제 → 해당 항목 제거, 총 금액 업데이트
- "전체 삭제" → 확인 모달("장바구니를 비우시겠습니까?") → 확인 시 장바구니 비우고 메뉴 화면 복귀
- "더 담기" → 메뉴 화면 (1-2) 복귀, 장바구니 유지
- "결제하기" → 결제 처리 (1-5)

**다음 단계:**
- 결제하기 → 결제 처리 화면 (1-5)
- 더 담기 → 메뉴 화면 (1-2)

**엣지 케이스:**
- **장바구니 비어있음**: "장바구니가 비어있습니다" 메시지, "결제하기" 버튼 비활성화
- **수량 0으로 변경**: 해당 항목 자동 삭제
- **장바구니 마지막 항목 삭제**: 장바구니가 비어있는 상태로 전환

---

### 1-5. 결제

**사용자가 보는 것:**
- 주문 요약 (총 금액)
- 토스페이먼츠 결제 위젯 (카드 결제)
- "결제 중입니다. 카드를 리더기에 접촉해 주세요." 안내 메시지
- 결제 진행 중: 로딩 스피너 + "결제 처리 중..." 메시지
- "주문 취소" 버튼

**가능한 동작:**
- 카드 결제 진행 → 성공 시 주문 완료 화면 (1-6)
- "주문 취소" → 확인 모달 → 확인 시 장바구니 화면 (1-4)으로 복귀

**다음 단계:**
- 결제 성공 → 주문 완료 화면 (1-6)
- 결제 실패 → 실패 안내 + 재시도/취소 선택

**엣지 케이스:**
- **결제 실패 (잔액 부족, 카드 오류)**: "결제에 실패했습니다." + 에러 사유 표시. "다시 시도" / "주문 취소" 버튼
- **결제 타임아웃 (30초 이상 무응답)**: "결제 시간이 초과되었습니다." 메시지. 자동으로 장바구니 화면 복귀
- **네트워크 끊김**: "네트워크 연결을 확인해 주세요." 메시지. 연결 복구 시 자동 재시도 또는 수동 재시도
- **결제는 승인됐지만 주문 생성 실패**: 결제 자동 취소(환불) 시도. 실패 시 점주에게 알림 (어드민에 수동 확인 필요 표시)

---

### 1-6. 주문 완료

**사용자가 보는 것:**
- 주문번호 크게 표시 (예: "A-042")
- "주문이 완료되었습니다!" 메시지
- 주문 내역 요약 (상품 목록, 총 금액)
- 자동 복귀 카운트다운 (예: "10초 후 처음 화면으로 돌아갑니다")

**가능한 동작:**
- 화면 터치 → 즉시 대기 화면 복귀 (카운트다운 종료 전에도 가능)
- 카운트다운 완료 → 자동으로 대기 화면 (1-1) 복귀

**다음 단계:**
- 대기 화면 (1-1)

**엣지 케이스:**
- **주문번호 생성 실패**: 결제는 완료됐으므로 "주문이 접수되었습니다. 주문번호는 카운터에서 확인해 주세요." 표시
- **카운트다운 중 다음 손님이 터치**: 즉시 새 주문 세션 시작 (이전 주문 데이터 완전 초기화)

## 2. 점주 플로우 (어드민)

### 2-1. 로그인

**사용자가 보는 것:**
- 로그인 폼: 이메일, 비밀번호
- "로그인" 버튼
- "비밀번호 찾기" 링크

**가능한 동작:**
- 이메일 + 비밀번호 입력 → 로그인 → 대시보드 (2-2)
- "비밀번호 찾기" → 비밀번호 재설정 이메일 발송

**엣지 케이스:**
- **잘못된 인증 정보**: "이메일 또는 비밀번호가 올바르지 않습니다." 에러 메시지
- **계정 잠김 (5회 실패)**: "계정이 잠겼습니다. 10분 후 다시 시도해 주세요." 메시지
- **이미 로그인 상태**: 대시보드로 리다이렉트
- **세션 만료**: 자동으로 로그인 페이지 리다이렉트, "세션이 만료되었습니다." 메시지

---

### 2-2. 대시보드

**사용자가 보는 것:**
- **오늘의 매출 요약**: 총 매출액, 주문 건수, 평균 주문 금액
- **최근 주문 목록** (최신 10건): 주문번호, 시간, 금액, 상태(대기/완료/취소)
- **오늘의 인기 상품** TOP 5: 상품명, 판매 수량
- **매장 상태**: 영업 중/영업 종료, 키오스크 연결 상태

**가능한 동작:**
- 주문 클릭 → 주문 상세 모달 (주문 항목, 결제 정보, 상태 변경)
- 주문 상태 변경: "대기" → "완료" 또는 "취소" (취소 시 환불 처리 확인)
- 좌측 네비게이션으로 다른 메뉴 이동

**엣지 케이스:**
- **오늘 주문이 없음**: "아직 오늘의 주문이 없습니다." 빈 상태 표시
- **매장 미설정 상태 (최초 가입)**: 온보딩 가이드 표시. "매장 정보를 먼저 설정해 주세요."

---

### 2-3. 상품 관리 — 카테고리

**사용자가 보는 것:**
- 카테고리 목록: 이름, 표시 순서, 소속 상품 수, 상태(활성/비활성)
- "카테고리 추가" 버튼
- 드래그 앤 드롭으로 순서 변경

**가능한 동작:**
- "카테고리 추가" → 카테고리 생성 모달 (이름, 표시 순서)
- 카테고리 편집 → 이름, 순서 변경
- 카테고리 비활성화 → 키오스크에서 해당 카테고리와 소속 상품 숨김
- 카테고리 삭제 → 소속 상품이 있으면 삭제 불가. "상품을 먼저 이동하거나 삭제해 주세요." 안내

**엣지 케이스:**
- **카테고리 이름 중복**: "이미 같은 이름의 카테고리가 있습니다." 에러
- **카테고리 0개 상태**: "카테고리를 추가하면 키오스크에 메뉴가 표시됩니다." 안내

---

### 2-4. 상품 관리 — 상품 목록

**사용자가 보는 것:**
- 상품 목록 테이블: 이미지(썸네일), 상품명, 카테고리, 가격, 상태(판매중/품절/숨김), 등록일
- 필터: 카테고리 선택, 상태 필터 (전체/판매중/품절/숨김)
- 검색: 상품명으로 검색
- "상품 추가" 버튼
- 각 행에 편집, 복제, 삭제 액션

**가능한 동작:**
- "상품 추가" → 상품 등록 화면 (2-5)
- 상품 편집 → 상품 수정 화면 (2-5)
- 상품 복제 → 동일 정보로 새 상품 생성 (이름 뒤에 "(복제)" 추가)
- 상품 삭제 → 확인 모달. 이미 주문에 포함된 상품은 소프트 삭제 (숨김 처리, 기존 주문 내역에는 유지)
- 상태 토글: 판매중 ↔ 품절 빠른 전환 (목록에서 바로)

**엣지 케이스:**
- **상품 0개**: "첫 번째 상품을 등록해 보세요!" 빈 상태 + CTA 버튼
- **검색 결과 없음**: "검색 결과가 없습니다." 메시지

---

### 2-5. 상품 관리 — 상품 등록/수정

**사용자가 보는 것:**
- **기본 정보**: 상품명(필수), 가격(필수), 설명(선택), 카테고리 선택(필수)
- **이미지**: 이미지 업로드 영역 (드래그 앤 드롭 또는 파일 선택), 최대 1장, 미리보기
- **옵션 관리**:
  - 옵션 그룹 추가 (예: "사이즈", "토핑")
  - 각 그룹: 그룹명, 필수/선택 여부, 최대 선택 개수 (선택 옵션인 경우)
  - 각 그룹 내 옵션 항목: 항목명, 추가 금액
- **상태**: 판매중 / 품절 / 숨김
- "저장" / "취소" 버튼

**가능한 동작:**
- 폼 입력 후 "저장" → 유효성 검증 → 성공 시 상품 목록으로 이동
- 이미지 업로드 → 미리보기 표시, 교체/삭제 가능
- 옵션 그룹 추가/삭제, 옵션 항목 추가/삭제
- "취소" → 변경사항 폐기 확인 모달 → 상품 목록으로 복귀

**유효성 검증:**
- 상품명: 필수, 1~50자
- 가격: 필수, 0원 이상의 정수 (100원 단위 권장이지만 강제 아님)
- 카테고리: 필수
- 이미지: 선택, 최대 5MB, JPG/PNG/WebP
- 옵션 그룹명: 필수 (그룹 존재 시), 1~20자
- 옵션 항목명: 필수, 1~30자
- 옵션 추가 금액: 0원 이상

**엣지 케이스:**
- **동일 카테고리 내 상품명 중복**: 허용 (경고만 표시)
- **이미지 업로드 실패**: "이미지 업로드에 실패했습니다. 다시 시도해 주세요." 에러. 상품 저장은 이미지 없이 가능
- **수정 중 다른 탭에서 같은 상품 수정**: 나중에 저장한 내용으로 덮어쓰기 (v1에서는 충돌 감지 미지원)

### 2-6. 주문 내역 조회

**사용자가 보는 것:**
- 주문 목록 테이블: 주문번호, 주문일시, 상품 요약(첫 상품 + 외 N건), 총 금액, 결제 상태, 주문 상태
- 필터:
  - 날짜 범위 (오늘/이번 주/이번 달/직접 입력)
  - 주문 상태 (전체/대기/완료/취소)
- 주문번호 검색
- 페이지네이션

**가능한 동작:**
- 주문 행 클릭 → 주문 상세 모달:
  - 주문 항목 목록 (상품명, 옵션, 수량, 소계)
  - 결제 정보 (결제 수단, 결제 시간, 승인번호)
  - 주문 상태 변경 버튼
- 주문 상태 변경: "대기" → "완료" (조리 완료 시), "대기" → "취소" (취소 사유 입력 + 환불)
- CSV 다운로드

**엣지 케이스:**
- **주문 취소 시 환불**: 토스페이먼츠 환불 API 호출. 환불 실패 시 "환불 처리에 실패했습니다. 토스페이먼츠 대시보드에서 수동 환불해 주세요." 안내
- **이미 완료된 주문 취소**: 완료 상태에서도 취소(환불) 가능 (당일 한정)

### 2-7. 매출 리포트

**사용자가 보는 것:**
- **기간 선택**: 일별 / 주별 / 월별 탭
- **매출 차트**: 막대 그래프 (기간별 매출 추이)
- **요약 카드**: 총 매출, 총 주문 수, 평균 주문 금액, 취소/환불 금액
- **상품별 판매 순위** TOP 10: 상품명, 판매 수량, 매출액
- **카테고리별 매출 비율**: 원형 차트

**가능한 동작:**
- 기간 변경 → 차트, 요약, 순위 업데이트
- 날짜 범위 직접 지정 (커스텀 기간)
- 리포트 CSV 다운로드

**엣지 케이스:**
- **데이터 없는 기간**: "해당 기간에 매출 데이터가 없습니다." 빈 차트
- **취소된 주문**: 매출 합계에서 제외, 별도로 취소/환불 금액 표시

### 2-8. 매장 설정

**사용자가 보는 것:**
- **매장 정보**: 매장명, 사업자등록번호, 대표자명, 연락처, 주소
- **영업시간**: 요일별 영업시간 설정 (시작 시간, 종료 시간, 휴무일 체크)
- **키오스크 설정**:
  - 대기 화면 메시지 (커스터마이징 가능)
  - 자동 복귀 타임아웃 (기본 10초, 5~30초 범위)
  - 비활성 타임아웃 (기본 5분, 1~10분 범위)
  - 주문번호 접두사 (기본 "A", 알파벳 1자)
  - 주문번호 일일 리셋 여부 (매일 001부터 시작)
- "저장" 버튼

**가능한 동작:**
- 각 설정 값 변경 후 "저장"
- 영업시간 변경 → 즉시 키오스크에 반영 (영업 시간 외 진입 차단)

**유효성 검증:**
- 매장명: 필수, 1~50자
- 연락처: 전화번호 형식
- 영업 시작 시간 < 종료 시간
- 타임아웃 값: 허용 범위 내

**엣지 케이스:**
- **영업시간 변경 중 키오스크에서 주문 진행 중**: 현재 진행 중인 주문은 영향 없음. 다음 주문부터 적용
- **모든 요일 휴무 설정**: 경고 메시지 "모든 요일이 휴무로 설정되어 키오스크가 작동하지 않습니다."
