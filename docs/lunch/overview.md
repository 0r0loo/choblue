# 점심 모집 앱 - 프로젝트 개요

## 프로젝트 비전

> "오늘 뭐 먹지?"를 "오늘 같이 먹자!"로 바꾸는 직장인 점심 모집 서비스

---

## 문제 정의

### 현재 상황

직장인들이 점심을 같이 먹을 사람을 모을 때 다음과 같은 방법을 쓴다:

1. **카카오톡 단체방**: "오늘 짬뽕 먹을 사람?" 메시지를 보낸다
   - 다른 대화에 묻혀서 못 보는 사람이 생긴다
   - 누가 참여하는지 파악이 안 된다 (읽씹인지 고민 중인지 불참인지)
   - 지난 기록을 찾기 어렵다
2. **슬랙/팀즈 채널**: 업무 메시지 사이에 점심 메시지가 섞인다
   - 점심 관련 대화만 필터링할 수 없다
   - 이모지 반응으로 참여 의사를 표시하지만 정확한 인원 파악이 번거롭다
3. **직접 돌아다니며 물어보기**: 자리에 없는 사람은 놓친다

### 핵심 불편함

- **참여 현황 파악이 안 된다**: 누가 가는지, 몇 명인지 한눈에 안 보인다
- **기록이 안 남는다**: 지난주에 누구랑 뭘 먹었는지 모른다
- **모집과 일반 대화가 섞인다**: 점심 모집글만 따로 볼 수 없다
- **시간 관리가 안 된다**: 마감 시간 없이 흐지부지 되거나, 이미 출발한 뒤에 참여 의사를 밝히는 경우가 생긴다

---

## 해결 방안

점심 모집 전용 서비스를 만든다.

- **모집글 단위로 관리**: 메뉴, 시간, 장소, 인원을 구조화된 폼으로 작성
- **참여 현황 실시간 표시**: 누가 참여했는지, 남은 자리가 몇 명인지 즉시 확인
- **달력 기반 UI**: 날짜별로 모집글을 한눈에 확인
- **자동 마감**: 시간 도래 또는 인원 충족 시 자동으로 마감
- **알림**: 새 모집글, 인원 변동, 마감 임박 등을 인앱 알림으로 전달
- **워크스페이스 분리**: 회사/팀/동호회 단위로 독립된 공간 제공
- **로그인 없이 바로 사용**: 링크 하나로 즉시 참여 가능

## 사용자 식별 방식 (MVP)

MVP에서는 계정 시스템 대신 **경량 식별 방식**을 사용한다:

1. **워크스페이스 생성 시**: 닉네임 입력 -> 브라우저에 토큰 저장 -> 관리용 비밀 링크 발급
2. **초대 링크로 참여 시**: 링크 클릭 -> 닉네임 입력 -> 브라우저에 토큰 저장 -> 바로 사용
3. **재방문 시**: 저장된 토큰으로 자동 인식, 닉네임 유지
4. **다른 기기/브라우저**: 새로운 참여자로 취급 (동일인 매칭 불가 -- MVP 한계로 수용)

이 방식의 장점:
- 슬랙/카톡에 링크 하나 던지면 **30초 안에** 사용 시작 가능
- 가입 과정 없이 즉시 체험 -> 유저 유입 장벽 제로

이 방식의 한계 (v2에서 해결):
- 브라우저 데이터 삭제 시 신원 소실
- 다기기 동기화 불가
- 악의적 사용자 차단 어려움

## 사용자 정의

### 주 사용자

| 페르소나 | 설명 |
|----------|------|
| **모집자** | 점심 메뉴를 정하고 같이 먹을 사람을 모으는 사람. 주로 점심 전 (10~11시) 모집글을 올린다. |
| **참여자** | 모집글을 보고 마음에 드는 메뉴/그룹에 참여하는 사람. 빠르게 확인하고 버튼 하나로 참여한다. |
| **워크스페이스 생성자** | 워크스페이스를 만들고 초대 링크를 공유하는 사람. 보통 팀장이나 총무 역할. |

### 사용자 특성

- 직장인 (20대 후반 ~ 40대)
- 점심시간이 정해져 있다 (보통 12:00~13:00)
- 모바일로 확인할 가능성이 높다 (자리에서 PC로 보는 경우도 있음)
- 빠르게 확인하고 빠르게 결정한다 (30초 이내에 참여 여부 결정)
- 새로운 앱에 회원가입하는 것을 귀찮아한다

## 핵심 유저 시나리오

### 시나리오 1: 워크스페이스 만들고 팀에 공유

> 박총무

1. 웹사이트에 접속해서 "워크스페이스 만들기" 클릭
2. 워크스페이스 이름 "ABC테크 개발팀", 본인 닉네임 "박총무" 입력
3. 워크스페이스가 생성되고 **초대 링크**와 **관리 링크**가 표시됨
4. 초대 링크를 복사해서 팀 슬랙에 "점심 모집 앱 만들었어요! 이 링크로 들어오세요" 공유
5. 관리 링크는 본인이 북마크 (워크스페이스 설정 변경용)

### 시나리오 2: 짬뽕 먹을 사람 모집

> 김개발 (모집자)

1. 오전 10:30, 점심 메뉴로 짬뽕이 땡긴다
2. 앱을 열고 오늘 날짜에 모집글을 작성한다
   - 메뉴: "짬뽕"
   - 장소: "회사 앞 만리장성"
   - 시간: 12:00
   - 최대인원: 4명
   - 메모: "탕수육 시켜서 나눠먹어요"
3. 모집글이 피드에 올라간다
4. 11:30까지 3명이 참여한다
5. 12:00에 4명이서 만리장성으로 출발한다

### 시나리오 3: 오늘 뭐 먹지? 둘러보기

> 이디자인 (참여자)

1. 오전 11:00, 앱을 연다
2. 오늘 날짜에 모집글이 3개 올라와 있다
   - 짬뽕 (2/4명) - 12:00
   - 샐러드 (1/3명) - 12:30
   - 돈까스 (3/4명) - 12:00
3. 짬뽕 모집글을 눌러 상세를 본다 (장소, 메모, 참여자 목록)
4. "참여" 버튼을 누른다
5. 참여 현황이 3/4명으로 업데이트된다

### 시나리오 4: 참여 취소

> 최마케팅 (참여자)

1. 오전에 돈까스 모집에 참여했다
2. 11:30에 갑자기 외부 미팅이 잡힌다
3. 앱에서 돈까스 모집글을 열고 "참여 취소"를 누른다
4. 참여 현황이 업데이트된다

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | Next.js (App Router), React 19, TypeScript |
| 백엔드 | NestJS, TypeScript |
| 데이터베이스 | PostgreSQL + TypeORM |
| 인증 | 없음 (MVP). 쿠키 기반 토큰으로 사용자 식별 |
| 스타일링 | Tailwind CSS v4 |
| UI 컴포넌트 | @choblue/ui (공유 디자인 시스템) |
| 실시간 | Polling 또는 Server-Sent Events (MVP는 단순하게) |
| 배포 | Vercel (프론트) + 별도 서버 (API) |
| 테스트 | Vitest, Playwright |
