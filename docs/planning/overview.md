# 프로젝트 개요

## 프로젝트 비전

소규모 매장(카페, 식당)이 별도 하드웨어 없이 iPad 한 대로 무인 주문 키오스크를 운영하고, 웹 어드민에서 메뉴와 매출을 관리할 수 있는 올인원 POS 시스템.

---

## v1 범위 정의

### 포함 (In Scope)

| 영역 | 상세 |
|------|------|
| 키오스크 주문 | 카테고리 탐색 → 상품 선택 → 옵션 선택 → 장바구니 → 카드 결제 → 주문번호 발급 |
| 카드 결제 | 토스페이먼츠 연동, 카드 결제만 지원 |
| 상품 관리 | 카테고리 CRUD, 상품 CRUD, 옵션 그룹/옵션 항목 관리, 상품 이미지 업로드 |
| 주문 관리 | 주문 내역 조회 (날짜/상태 필터), 주문 상태 변경 (대기 → 완료/취소) |
| 매출 리포트 | 일별/주별/월별 매출 요약, 상품별 판매 순위 |
| 매장 설정 | 매장 정보, 영업시간, 키오스크 설정 (대기화면 메시지, 자동복귀 타임아웃) |
| 인증 | 점주 로그인 (Better Auth), 어드민 접근 제어 |

### 미포함 (Out of Scope — v2 이후)

- 테이블 오더 (QR 코드 기반 좌석 주문)
- 다점포 지원 (여러 매장을 하나의 계정으로 관리)
- 직원 계정 및 권한 분리
- 현금 결제, 간편결제(카카오페이, 네이버페이 등)
- 세트메뉴 구성
- 재고 관리
- 프린터/KDS(주방 디스플레이) 연동
- 쿠폰/할인/적립 시스템
- AI 매출 분석
- 푸시 알림 (신규 주문 알림)

## 사용자 정의

### 고객 (Customer)

- **누구**: 매장에 방문한 손님
- **사용 환경**: 카운터에 설치된 iPad 키오스크 (터치 인터랙션)
- **목표**: 빠르게 메뉴를 고르고, 카드로 결제하고, 주문번호를 받는다
- **특성**: 앱 설치 없음, 로그인 없음, 1회성 사용. 다양한 연령대가 사용하므로 직관적 UI 필수

### 점주 (Store Owner)

- **누구**: 매장을 운영하는 사장님 (1인 운영 또는 소규모)
- **사용 환경**: PC 또는 태블릿 웹 브라우저 (어드민)
- **목표**: 메뉴를 등록/수정하고, 매출을 확인하고, 매장 설정을 관리한다
- **특성**: IT 전문가가 아님. 복잡한 설정 없이 바로 쓸 수 있어야 한다

## 핵심 유저 시나리오

### 시나리오 1: 고객이 아메리카노를 주문한다

> 손님이 카운터에 놓인 iPad를 터치한다. 메뉴 화면이 나타나고 "커피" 카테고리를 누른다. 아메리카노를 선택하면 옵션 선택 모달이 뜬다. 사이즈를 "라지(+500원)"로 변경하고 "장바구니에 담기"를 누른다. 장바구니에서 수량을 확인하고 "결제하기"를 누르면 토스페이먼츠 결제 화면이 나타난다. 카드를 긁으면 결제가 완료되고 주문번호 "A-042"가 화면에 크게 표시된다. 10초 후 자동으로 대기 화면으로 돌아간다.

### 시나리오 2: 점주가 신메뉴를 등록한다

> 점주가 어드민에 로그인한다. "상품 관리" 메뉴에서 "디저트" 카테고리를 선택하고 "상품 추가"를 누른다. 상품명 "딸기 크로플", 가격 "6,500원", 설명 "신선한 딸기와 바삭한 크로플"을 입력한다. 이미지를 업로드하고 옵션 그룹 "토핑 추가"를 설정한다(생크림 +500원, 아이스크림 +1,000원). "저장"을 누르면 키오스크에 즉시 반영된다.

### 시나리오 3: 점주가 품절 상품을 처리한다

> 오후에 딸기가 떨어졌다. 점주가 어드민에서 "딸기 스무디" 상품의 상태를 "품절"로 변경한다. 키오스크 메뉴에서 해당 상품이 품절 표시되고 선택이 불가능해진다. 다음 날 딸기가 입고되면 다시 "판매중"으로 변경한다.

### 시나리오 4: 결제 중 오류가 발생한다

> 손님이 결제 버튼을 눌렀지만 카드 결제가 실패한다 (잔액 부족). 화면에 "결제에 실패했습니다. 다른 카드로 시도해 주세요." 메시지가 표시된다. 손님은 "다시 시도" 버튼을 눌러 다른 카드로 결제하거나, "주문 취소" 버튼을 눌러 대기 화면으로 돌아갈 수 있다.

## 기술 스택 요약

| 영역 | 기술 |
|------|------|
| 키오스크 프론트엔드 | Next.js 15 (App Router), React 19, TypeScript |
| 어드민 프론트엔드 | Next.js 15 (App Router), React 19, TypeScript |
| API 서버 | NestJS, TypeORM, PostgreSQL |
| 인증 | Better Auth |
| 결제 | 토스페이먼츠 (카드 결제 SDK + 웹훅) |
| 스타일링 | Tailwind CSS v4, CVA |
| UI 컴포넌트 | @choblue/ui (자체 디자인 시스템) |
| 상태 관리 | Zustand (클라이언트), TanStack Query (서버) |
| 폼 관리 | React Hook Form + Zod |
| 이미지 저장 | S3 호환 스토리지 (Cloudflare R2 또는 AWS S3) |
| 모노레포 | pnpm workspace + Turborepo |
| 테스트 | Vitest, Playwright, Storybook |
