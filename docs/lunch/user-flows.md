# 점심 모집 앱 - 유저 플로우

## 1. 온보딩 플로우

MVP에서는 로그인/회원가입이 없다. 두 가지 진입 경로가 있다.

---

### 1-A. 워크스페이스 생성 플로우

새로운 워크스페이스를 만드는 사람의 플로우.

```
랜딩 페이지 접속
    |
    v
"워크스페이스 만들기" 클릭
    |
    v
워크스페이스 정보 입력 폼
  - 워크스페이스 이름 (필수, 예: "ABC테크 개발팀")
  - 내 닉네임 (필수, 예: "박총무")
  - 워크스페이스 설명 (선택, 예: "개발팀 점심 모임")
    |
    v
워크스페이스 생성 완료 화면
  - 초대 링크 표시 + 복사 버튼
  - 관리 링크 표시 + 복사 버튼 (북마크 안내)
  - "시작하기" 버튼 -> 메인 화면으로 이동
```

#### 단계별 상세

| 단계 | 사용자가 보는 것 | 가능한 동작 | 다음 단계 |
|------|-----------------|------------|----------|
| 랜딩 페이지 | 서비스 소개, "만들기" / "참여하기" 버튼 | 워크스페이스 만들기 클릭 | 생성 폼 |
| 생성 폼 | 이름, 닉네임, 설명 입력 필드 | 입력 후 "만들기" 클릭 | 완료 화면 |
| 완료 화면 | 초대 링크, 관리 링크, 시작 버튼 | 링크 복사, 시작하기 클릭 | 메인 화면 |

#### 예외 상황
- 워크스페이스 이름 미입력: 폼 유효성 검사, 에러 메시지 표시
- 닉네임 미입력: 폼 유효성 검사, 에러 메시지 표시
- 네트워크 오류: "다시 시도해주세요" 메시지

#### 기술 구현 노트
- 워크스페이스 생성 시 서버에서 고유 토큰 발급 -> 브라우저 쿠키에 저장
- 초대 링크: `/{workspace-slug}/invite/{invite-code}`
- 관리 링크: `/{workspace-slug}/admin/{admin-token}` (워크스페이스 설정 접근용)

---

### 1-B. 초대 링크로 참여 플로우

누군가가 공유한 초대 링크를 통해 들어오는 사람의 플로우.

```
초대 링크 클릭 (슬랙, 카톡 등에서)
    |
    v
참여 화면
  - 워크스페이스 이름 표시 ("ABC테크 개발팀에 참여합니다")
  - 닉네임 입력 (필수)
    |
    v
"참여하기" 클릭
    |
    v
메인 화면 (해당 워크스페이스)
```

#### 단계별 상세

| 단계 | 사용자가 보는 것 | 가능한 동작 | 다음 단계 |
|------|-----------------|------------|----------|
| 초대 링크 접속 | 워크스페이스 이름, 닉네임 입력 필드 | 닉네임 입력 후 "참여하기" | 메인 화면 |

#### 예외 상황
- 유효하지 않은 초대 링크: "유효하지 않은 링크입니다" 메시지 + 랜딩 페이지 링크
- 닉네임 미입력: 폼 유효성 검사
- 이미 참여한 워크스페이스 (같은 브라우저): 바로 메인 화면으로 리다이렉트
- 닉네임 중복: 허용한다 (MVP에서는 동명이인 가능. 구분은 내부 토큰으로)

---

### 1-C. 재방문 플로우

이미 워크스페이스에 참여한 사람이 다시 방문하는 경우.

```
앱 URL 접속 또는 북마크
    |
    v
쿠키 확인 -> 토큰 존재?
    |
    +--> 있음: 마지막 접속한 워크스페이스 메인 화면
    |
    +--> 없음: 랜딩 페이지 (새로운 방문자 취급)
```

#### 여러 워크스페이스 소속인 경우
- 쿠키에 여러 워크스페이스 토큰이 저장되어 있으면 **워크스페이스 선택 화면** 표시
- 하나만 있으면 바로 해당 워크스페이스 메인으로 이동

---

## 2. 점심 모집 플로우 (모집자)

모집글을 작성하고 관리하는 플로우.

### 2-A. 모집글 작성

```
메인 화면 (달력 뷰)
    |
    v
"모집하기" 버튼 클릭 (FAB 또는 헤더)
    |
    v
모집글 작성 폼
  - 메뉴명 (필수, 텍스트. 예: "짬뽕")
  - 식당/장소 (선택, 텍스트. 예: "회사 앞 만리장성")
  - 날짜 (기본값: 오늘, 변경 가능)
  - 시간 (기본값: 12:00, 30분 단위 선택)
  - 최대인원 (기본값: 4명, 2~10명 범위)
  - 메모 (선택, 텍스트. 예: "탕수육 시켜서 나눠먹어요")
    |
    v
"올리기" 클릭
    |
    v
모집글 상세 화면 (방금 만든 글)
  - 모집자 자신은 자동으로 참여 처리됨 (1/N명)
```

#### 단계별 상세

| 단계 | 사용자가 보는 것 | 가능한 동작 | 다음 단계 |
|------|-----------------|------------|----------|
| 메인 화면 | 달력, 오늘 모집글 목록, 모집하기 버튼 | "모집하기" 클릭 | 작성 폼 |
| 작성 폼 | 입력 필드들, 올리기/취소 버튼 | 필드 입력, 올리기 또는 취소 | 상세 화면 또는 메인 |
| 상세 화면 | 모집 정보, 참여자 목록 (본인 포함), 공유/수정/삭제 버튼 | 공유, 수정, 삭제, 마감 | - |

#### 예외 상황
- 메뉴명 미입력: 폼 유효성 검사, "메뉴를 입력해주세요"
- 과거 날짜 선택: 선택 불가 (달력에서 과거 날짜 비활성화)
- 이미 지난 시간 선택: "이미 지난 시간입니다" 경고 (오늘 날짜인 경우)

---

### 2-B. 모집글 수정

```
모집글 상세 화면
    |
    v
"수정" 버튼 클릭 (모집자만 보임)
    |
    v
수정 폼 (기존 값이 채워진 상태)
    |
    v
"수정하기" 클릭
    |
    v
수정된 상세 화면
```

#### 수정 가능 조건
- 모집자 본인만 수정 가능
- 마감된 모집글은 수정 불가
- 최대인원을 현재 참여인원보다 적게 줄일 수 없음

---

### 2-C. 모집글 삭제

```
모집글 상세 화면
    |
    v
"삭제" 버튼 클릭
    |
    v
확인 다이얼로그
  "모집글을 삭제하면 참여자들의 참여도 취소됩니다. 삭제하시겠습니까?"
    |
    +--> "삭제" -> 모집글 삭제, 메인 화면으로 이동
    |
    +--> "취소" -> 다이얼로그 닫기
```

---

### 2-D. 모집 마감

```
모집글 상세 화면
    |
    v
마감 방법 2가지:
    |
    +--> 자동 마감: 설정한 시간이 되면 자동 마감
    |    또는: 최대인원이 충족되면 자동 마감
    |
    +--> 수동 마감: "마감하기" 버튼 클릭 (모집자만)
         -> 확인 다이얼로그 -> 마감 처리
    |
    v
마감 상태 표시
  - "마감됨" 뱃지
  - 참여/취소 버튼 비활성화
  - 최종 참여자 목록 표시
```

---

## 3. 참여 플로우 (참여자)

모집글을 보고 참여하는 플로우.

### 3-A. 모집글 확인 및 참여

```
메인 화면 (달력 뷰 또는 오늘 피드)
    |
    v
모집글 카드 확인
  - 메뉴명, 시간, 현재인원/최대인원, 모집자 닉네임
    |
    v
모집글 카드 클릭 -> 상세 화면
    |
    v
상세 화면에서 정보 확인
  - 메뉴, 장소, 시간, 메모, 참여자 목록
    |
    v
"참여하기" 버튼 클릭
    |
    v
참여 완료
  - 버튼이 "참여 취소"로 변경
  - 참여자 목록에 내 닉네임 추가
  - 인원 카운트 업데이트
```

#### 단계별 상세

| 단계 | 사용자가 보는 것 | 가능한 동작 | 다음 단계 |
|------|-----------------|------------|----------|
| 메인 피드 | 오늘 모집글 카드 목록 | 카드 클릭 | 상세 화면 |
| 상세 화면 | 모집 상세 정보, 참여자 목록, 참여 버튼 | 참여하기 클릭 | 참여 완료 상태 |
| 참여 완료 | 참여 취소 버튼, 업데이트된 참여자 목록 | 참여 취소 가능 | - |

#### 예외 상황
- 이미 마감된 모집글: "참여하기" 버튼 비활성화, "마감되었습니다" 표시
- 인원이 꽉 찬 모집글: "참여하기" 버튼 비활성화, "인원이 마감되었습니다" 표시
- 동시 참여로 인원 초과: 서버에서 거부, "인원이 마감되었습니다" 메시지

---

### 3-B. 참여 취소

```
모집글 상세 화면 (참여 중인 상태)
    |
    v
"참여 취소" 버튼 클릭
    |
    v
확인 다이얼로그
  "참여를 취소하시겠습니까?"
    |
    +--> "취소하기" -> 참여 취소, 버튼이 "참여하기"로 변경
    |
    +--> "아니요" -> 다이얼로그 닫기
```

#### 예외 상황
- 마감된 모집글에서 참여 취소: 불가. "마감된 모집은 취소할 수 없습니다" 메시지

---

## 4. 달력 뷰 플로우

날짜별 모집글을 탐색하는 플로우.

```
메인 화면
    |
    v
달력 표시 (월간 뷰, 기본)
  - 각 날짜 셀에 모집글 개수 표시 (점 또는 숫자)
  - 오늘 날짜 강조
    |
    v
날짜 클릭
    |
    v
해당 날짜의 모집글 목록 표시 (달력 아래)
  - 각 모집글 카드: 메뉴, 시간, 인원, 상태(모집중/마감)
    |
    v
이전/다음 달 네비게이션
```

#### 달력 날짜 셀 표시 규칙
- 모집글 없음: 빈 셀
- 모집글 있음: 점(dot) 표시 (모집중: 컬러, 전부 마감: 회색)
- 오늘: 배경색 강조
- 과거: 연한 색상 (모집 불가 표시)
- 미래: 일반 색상 (모집 가능)

---

## 5. 워크스페이스 관리 플로우

### 5-A. 초대 링크 공유

```
메인 화면 -> 사이드바 또는 설정 아이콘
    |
    v
워크스페이스 설정 화면
    |
    v
"초대 링크" 섹션
  - 현재 초대 링크 표시
  - "복사" 버튼 -> 클립보드에 복사, 토스트 메시지 "복사되었습니다"
  - "링크 재발급" 버튼 (기존 링크 무효화)
```

---

### 5-B. 멤버 목록 조회

```
워크스페이스 설정 화면
    |
    v
"멤버" 섹션
  - 멤버 목록 (닉네임, 참여일)
  - 총 멤버 수 표시
```

#### 제한 사항 (MVP)
- 멤버 강제 탈퇴 기능 없음 (v2)
- 역할(관리자/멤버) 구분 없음 (생성자만 관리 링크 보유)

---

### 5-C. 워크스페이스 설정 변경

관리 링크를 가진 사람(생성자)만 접근 가능.

```
관리 링크 접속 또는 설정 화면 (관리자인 경우)
    |
    v
워크스페이스 설정 폼
  - 워크스페이스 이름 변경
  - 설명 변경
  - 초대 링크 재발급
```

---

## 6. 프로필 설정 플로우

### 6-A. 닉네임 변경

```
설정 아이콘 또는 프로필 영역 클릭
    |
    v
프로필 설정 화면
  - 현재 닉네임 표시
  - 닉네임 변경 입력 필드
  - "저장" 버튼
    |
    v
닉네임 변경 완료 -> 토스트 메시지 "닉네임이 변경되었습니다"
```

## 7. 전체 플로우 요약 다이어그램

```
[랜딩 페이지]
    |
    +-- "만들기" --> [워크스페이스 생성] --> [완료 (링크 발급)] --> [메인]
    |
    +-- 초대 링크 --> [닉네임 입력] --> [메인]
    |
    +-- 재방문 (쿠키) --> [워크스페이스 선택*] --> [메인]

[메인 화면]
    |
    +-- 달력 날짜 클릭 --> [해당 날짜 모집글 목록]
    |
    +-- "모집하기" --> [작성 폼] --> [모집글 상세]
    |
    +-- 모집글 카드 클릭 --> [모집글 상세]
    |       |
    |       +-- "참여하기" / "참여 취소"
    |       +-- "수정" / "삭제" / "마감" (모집자)
    |
    +-- 설정 --> [워크스페이스 설정]
    |              |
    |              +-- 초대 링크
    |              +-- 멤버 목록
    |              +-- 프로필 (닉네임)
    |
    * 워크스페이스가 여러 개일 때만 표시
```
