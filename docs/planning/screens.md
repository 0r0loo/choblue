# 화면 명세

---

## 키오스크 화면

### K-1. 대기 화면 (Idle Screen)

| 항목 | 내용 |
|------|------|
| **경로** | `/kiosk` |
| **목적** | 손님의 주문 시작을 유도하는 대기 상태 화면 |

**주요 구성요소:**
- 매장 로고 이미지 (중앙 상단)
- 환영 메시지 텍스트 (점주가 설정 가능, 기본값: "주문하시려면 화면을 터치해 주세요")
- 배경 이미지 또는 그라데이션
- 하단 영업시간 표시 (선택적)

**데이터:**
- 매장 정보: 매장명, 로고, 환영 메시지
- 영업 상태: 현재 영업 중 여부
- 휴무 상태

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 화면 아무 곳 터치 | 영업 중이면 메뉴 화면(K-2)으로 전환. 전환 시 페이드 애니메이션 |
| 영업시간 외 터치 | 반응 없음. 영업시간 안내 메시지 유지 |

### K-2. 메뉴 화면 (Menu Screen)

| 항목 | 내용 |
|------|------|
| **경로** | `/kiosk/menu` |
| **목적** | 카테고리별 상품을 탐색하고 장바구니에 담는 메인 주문 화면 |

**주요 구성요소:**
- **상단 헤더**: 매장명, "처음으로" 버튼
- **좌측 사이드바 (너비 ~200px)**: 카테고리 탭 목록 (세로 나열, 선택된 카테고리 하이라이트)
- **중앙 콘텐츠 (메인 영역)**: 상품 그리드 (2~3열)
  - 각 상품 카드: 이미지 (정사각형 썸네일), 상품명, 가격, 품절 시 품절 오버레이
- **우측 사이드바 (너비 ~300px)**: 장바구니 미니 패널
  - 담긴 상품 목록 (상품명, 옵션 요약, 수량, 소계)
  - 총 합계 금액
  - "결제하기" 버튼 (금액 포함, 예: "15,000원 결제하기")
  - 장바구니 비어있으면: "메뉴를 선택해 주세요" 안내

**데이터:**
- 카테고리 목록 (활성 카테고리만, 표시 순서대로)
- 선택된 카테고리의 상품 목록 (활성 상품만, 표시 순서대로)
- 각 상품: id, 이름, 가격, 이미지 URL, 품절 여부, 옵션 유무
- 장바구니 상태: 담긴 아이템 배열 (상품 + 선택 옵션 + 수량)

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 카테고리 탭 터치 | 해당 카테고리 상품으로 그리드 전환 (애니메이션 없이 즉시) |
| 상품 카드 터치 | 옵션 선택 모달(K-3) 열기 |
| 품절 상품 카드 터치 | 반응 없음 (터치 이벤트 무시) |
| 장바구니 항목 수량 +/- | 수량 및 금액 즉시 업데이트 |
| 장바구니 항목 삭제 (X) | 해당 항목 제거 |
| "결제하기" 버튼 | 장바구니/결제 화면(K-4)으로 이동 |
| "처음으로" 버튼 | 장바구니에 항목이 있으면 확인 모달, 확인 시 장바구니 초기화 + 대기 화면(K-1) |
| 5분간 미조작 | 비활성 경고 모달 → 15초 카운트다운 → 장바구니 초기화 + 대기 화면 |

### K-3. 옵션 선택 모달 (Option Selection Modal)

| 항목 | 내용 |
|------|------|
| **경로** | `/kiosk/menu` (모달 오버레이, URL 변경 없음) |
| **목적** | 상품의 옵션(사이즈, 토핑 등)과 수량을 선택 |

**주요 구성요소:**
- **모달 배경**: 반투명 딤 처리
- **모달 컨테이너** (화면 중앙, 너비 ~600px):
  - 상품 이미지 (큰 사이즈, 상단)
  - 상품명, 기본 가격
  - 상품 설명 (있는 경우)
  - 구분선
  - **옵션 그룹 영역** (스크롤 가능):
    - 그룹 헤더: 그룹명 + "필수" 또는 "선택" 뱃지
    - 필수 옵션: 라디오 버튼 리스트 (항목명 + 추가 금액)
    - 선택 옵션: 체크박스 리스트 (항목명 + 추가 금액)
  - 구분선
  - **수량 조절**: [-] 1 [+] (크기 44px 이상의 터치 타겟)
  - **합계 금액**: "총 5,500원" (실시간 업데이트)
  - **하단 버튼 영역**: "취소" (좌), "장바구니에 담기 · 5,500원" (우, 강조 색상)
- 우측 상단 X 닫기 버튼

**데이터:**
- 상품 상세: id, 이름, 가격, 설명, 이미지 URL
- 옵션 그룹 배열:
  - 그룹명, 필수 여부, 최대 선택 수
  - 옵션 항목 배열: 항목명, 추가 금액

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 필수 옵션 항목 선택 | 라디오 전환, 합계 금액 즉시 업데이트 |
| 선택 옵션 항목 토글 | 체크/해제, 합계 금액 즉시 업데이트 |
| 선택 옵션 최대 개수 초과 | 토스트: "최대 N개까지 선택 가능합니다." 추가 선택 무시 |
| 수량 + 터치 | 수량 증가, 합계 업데이트 |
| 수량 - 터치 (수량 1일 때) | 반응 없음 (버튼 비활성화 스타일) |
| "장바구니에 담기" | 장바구니에 추가, 모달 닫힘, 메뉴 화면 장바구니 패널 업데이트 |
| "취소" / X / 딤 영역 터치 | 모달 닫힘, 변경사항 없음 |

### K-4. 장바구니/결제 화면 (Cart & Payment Screen)

| 항목 | 내용 |
|------|------|
| **경로** | `/kiosk/cart` |
| **목적** | 주문 내역을 최종 확인하고 결제를 진행 |

**주요 구성요소:**
- **상단 헤더**: "주문 확인", "뒤로" 버튼
- **주문 내역 리스트** (스크롤 가능):
  - 각 항목: 상품명, 선택 옵션 (태그 형태), 수량 조절 [-/+], 소계 금액, 삭제 버튼(X)
- **주문 요약 영역**:
  - 총 수량: "총 N개"
  - 총 금액: "총 15,000원" (크게)
- **하단 버튼 영역**:
  - "더 담기" 버튼 (보조, 좌측) → 메뉴 화면 복귀
  - "결제하기 · 15,000원" 버튼 (강조, 우측)
- **전체 삭제** 링크/버튼 (상단 우측)

**데이터:**
- 장바구니 아이템 배열: 상품 정보 + 선택 옵션 + 수량 + 소계
- 총 합계 금액

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 수량 변경 | 소계, 총 금액 즉시 업데이트 |
| 항목 삭제 | 확인 없이 즉시 삭제, 금액 업데이트 |
| "전체 삭제" | 확인 모달 → 확인 시 장바구니 비우기, 메뉴 화면(K-2)으로 |
| "더 담기" | 메뉴 화면(K-2)으로, 장바구니 유지 |
| "결제하기" | 결제 처리 화면(K-5)으로 전환 |
| "뒤로" | 메뉴 화면(K-2)으로, 장바구니 유지 |
| 장바구니 비어있는 상태 | "결제하기" 버튼 비활성화, "메뉴를 선택해 주세요" 안내 |

### K-5. 결제 처리 중 화면 (Payment Processing Screen)

| 항목 | 내용 |
|------|------|
| **경로** | `/kiosk/payment` |
| **목적** | 토스페이먼츠 결제 진행 및 결과 처리 |

**주요 구성요소:**
- **결제 단계 표시**:
  - 단계 1: 토스페이먼츠 결제 위젯 (카드 정보 입력 또는 NFC 리더)
  - 단계 2: 결제 처리 중 로딩 (스피너 + "결제 처리 중입니다..." 메시지)
- **결제 금액 표시**: "결제 금액: 15,000원"
- **취소 버튼**: "주문 취소" (결제 처리 중에는 비활성화)

**결제 실패 상태 구성요소:**
- 실패 아이콘 (X 마크)
- 에러 메시지 ("결제에 실패했습니다.", 사유 표시)
- "다시 시도" 버튼
- "주문 취소" 버튼

**데이터:**
- 주문 정보: 장바구니 아이템, 총 금액
- 토스페이먼츠 결제 키 (paymentKey)
- 결제 상태: 대기 / 처리중 / 성공 / 실패

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 카드 결제 완료 (성공) | 주문 생성 API 호출 → 성공 시 주문 완료 화면(K-6) |
| 카드 결제 실패 | 실패 화면 표시 |
| "다시 시도" | 결제 위젯 재로드 |
| "주문 취소" | 확인 모달 → 확인 시 장바구니 화면(K-4) 복귀 |
| 30초 타임아웃 | 자동으로 실패 처리 + 안내 메시지 |

### K-6. 주문 완료 화면 (Order Complete Screen)

| 항목 | 내용 |
|------|------|
| **경로** | `/kiosk/complete` |
| **목적** | 주문 완료를 알리고 주문번호를 안내 |

**주요 구성요소:**
- 체크마크 아이콘 (성공 애니메이션)
- "주문이 완료되었습니다!" 텍스트 (크게)
- **주문번호** (매우 크게, 예: "A-042", 폰트 사이즈 72px 이상)
- 주문 요약: 상품명 나열, 총 금액
- 자동 복귀 카운트다운 바: "10초 후 처음 화면으로 돌아갑니다"
- 안내 문구: "주문번호가 호출되면 카운터에서 수령해 주세요"

**데이터:**
- 주문번호
- 주문 항목 요약
- 총 결제 금액
- 자동 복귀 타임아웃 (초)

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 화면 터치 | 즉시 대기 화면(K-1)으로 복귀 |
| 카운트다운 완료 | 자동으로 대기 화면(K-1)으로 복귀 |

## 어드민 화면

### A-1. 로그인 (Login)

| 항목 | 내용 |
|------|------|
| **경로** | `/admin/login` |
| **목적** | 점주 인증 |

**주요 구성요소:**
- 로고 + 서비스명
- 로그인 폼:
  - 이메일 입력 필드
  - 비밀번호 입력 필드 (비밀번호 표시/숨김 토글)
  - "로그인" 버튼
- "비밀번호 찾기" 링크
- 에러 메시지 영역 (인라인)

**데이터:**
- 입력값: 이메일, 비밀번호
- 에러 메시지

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 로그인 성공 | 대시보드(A-2)로 리다이렉트 |
| 로그인 실패 | 인라인 에러 메시지 표시 |
| Enter 키 | 로그인 폼 제출 |
| 비밀번호 찾기 클릭 | 비밀번호 재설정 페이지 이동 |

### A-2. 대시보드 (Dashboard)

| 항목 | 내용 |
|------|------|
| **경로** | `/admin` |
| **목적** | 오늘의 매출과 최근 주문을 한눈에 파악 |

**주요 구성요소:**
- **좌측 네비게이션** (모든 어드민 화면 공통):
  - 대시보드, 상품 관리, 주문 내역, 매출 리포트, 매장 설정
  - 하단: 점주 이름, 로그아웃
- **매출 요약 카드** (3개 가로 배치):
  - 오늘 총 매출 (금액 + 전일 대비 증감률)
  - 오늘 주문 건수 (건수 + 전일 대비 증감)
  - 평균 주문 금액
- **최근 주문** (테이블, 최신 10건):
  - 주문번호, 시간, 주문 내용 요약, 금액, 상태 뱃지
- **인기 상품 TOP 5** (리스트):
  - 순위, 상품명, 판매 수량

**데이터:**
- 오늘 매출 집계: 총 매출, 주문 수, 평균 금액, 전일 비교
- 최근 주문 10건: 주문번호, 시간, 상품 요약, 금액, 상태
- 인기 상품 TOP 5: 상품명, 판매 수량

**인터랙션:**
| 동작 | 결과 |
|------|------|
| 주문 행 클릭 | 주문 상세 모달 열기 (주문 항목 상세, 결제 정보, 상태 변경) |
| 주문 상태 변경 | "대기" → "완료" 또는 "취소" (취소 시 사유 입력 + 환불 확인) |
| 네비게이션 메뉴 클릭 | 해당 화면으로 이동 |

### A-3. 카테고리 관리 (Category Management)

| 항목 | 내용 |
|------|------|
| **경로** | `/admin/categories` |
| **목적** | 키오스크에 표시되는 메뉴 카테고리 관리 |

**주요 구성요소:**
- 페이지 제목: "카테고리 관리"
- "카테고리 추가" 버튼 (우측 상단)
- **카테고리 리스트** (드래그 앤 드롭 가능):
  - 드래그 핸들 (⠿ 아이콘)
  - 카테고리명
  - 소속 상품 수 (뱃지)
  - 상태: 활성/비활성 토글 스위치
  - 액션: 편집, 삭제 버튼
- 빈 상태: "카테고리를 추가하면 키오스크에 메뉴가 표시됩니다."

**카테고리 추가/편집 모달:**
- 카테고리명 입력 (필수, 1~30자)
- "저장" / "취소" 버튼

**데이터:**
- 카테고리 배열: id, 이름, 표시 순서, 소속 상품 수, 활성 상태

**인터랙션:**
| 동작 | 결과 |
|------|------|
| "카테고리 추가" 클릭 | 추가 모달 열기 |
| 드래그 앤 드롭 | 표시 순서 변경 (즉시 저장) |

### A-5. 상품 등록/수정 (Product Create/Edit)

| 항목 | 내용 |
|------|------|
| **경로** | `/admin/products/new` (등록) / `/admin/products/[id]/edit` (수정) |
| **목적** | 상품 정보와 옵션을 등록하거나 수정 |

**주요 구성요소:**
- 페이지 제목: "상품 등록" 또는 "상품 수정"
- **기본 정보 섹션**:
  - 상품명 입력 (필수)
  - 가격 입력 (필수, 숫자만)
  - 설명 텍스트에어리어 (선택, 최대 200자)
  - 카테고리 드롭다운 (필수)
  - 상태 라디오: 판매중 / 품절 / 숨김
- **이미지 섹션**:
  - 이미지 업로드 영역 (클릭 또는 드래그 앤 드롭)
  - 업로드 후: 미리보기 + 삭제(X) 버튼
  - 안내: "JPG, PNG, WebP / 최대 5MB"
- **옵션 관리 섹션**:
  - "옵션 그룹 추가" 버튼
  - 각 옵션 그룹 카드:
    - 그룹명 입력
    - 필수/선택 토글
    - 선택 옵션인 경우: 최대 선택 수 입력
    - 옵션 항목 리스트: 항목명 + 추가 금액 + 삭제 버튼
    - "항목 추가" 버튼
    - 그룹 삭제 버튼
- **하단 액션 영역**:
  - "취소" 버튼 (좌)
  - "저장" 버튼 (우, 강조)

**데이터:**
- (수정 시) 기존 상품 정보: 이름, 가격, 설명, 카테고리 id, 이미지 URL, 상태, 옵션 그룹/항목
- 카테고리 목록 (드롭다운용)

**인터랙션:**
| 동작 | 결과 |
|------|------|
| "저장" 클릭 | 유효성 검증 → 통과 시 API 호출 → 성공 시 상품 목록(A-4)으로 이동 + 성공 토스트 |
| 유효성 검증 실패 | 해당 필드에 인라인 에러 메시지, 첫 에러 필드로 스크롤 |
| 이미지 드래그 앤 드롭 | 미리보기 표시, 저장 시 업로드 |
| 이미지 5MB 초과 | "이미지 크기는 5MB 이하여야 합니다." 에러 |
| "취소" 클릭 | 변경사항 있으면 확인 모달 → 상품 목록(A-4)으로 |
| 옵션 그룹 추가 | 빈 옵션 그룹 카드 추가 |
| 옵션 항목 추가 | 그룹 내 빈 옵션 항목 행 추가 |

**데이터:**

- 주문 상세: 주문 항목 배열, 결제 정보, 상태 이력

**인터랙션:**
| 동작 | 결과 |

| "CSV 다운로드" | 현재 필터 조건의 주문 데이터 CSV 파일 다운로드 |

### A-8. 매장 설정 (Store Settings)

| 항목 | 내용 |
|------|------|
| **경로** | `/admin/settings` |
| **목적** | 매장 정보, 영업시간, 키오스크 설정 관리 |

**주요 구성요소:**
- 페이지 제목: "매장 설정"
- **매장 정보 섹션**:
  - 매장명 입력
  - 사업자등록번호 입력
  - 대표자명 입력
  - 연락처 입력
  - 주소 입력
- **영업시간 섹션**:
  - 요일별 행 (월~일):
    - 영업 체크박스 (해제 = 휴무)
    - 시작 시간 (TimePicker)
    - 종료 시간 (TimePicker)
  - "전체 동일 적용" 버튼 (첫 번째 행의 시간을 전체 적용)
- **키오스크 설정 섹션**:
  - 대기 화면 메시지 입력
  - 주문 완료 후 자동 복귀 시간 (슬라이더 또는 숫자 입력, 5~30초)
  - 비활성 타임아웃 (슬라이더 또는 숫자 입력, 1~10분)
  - 주문번호 접두사 (알파벳 1자, 기본 "A")
  - 주문번호 일일 리셋 토글 (기본 ON)
- "저장" 버튼

**데이터:**
- 매장 정보: 이름, 사업자번호, 대표자, 연락처, 주소
- 영업시간: 요일별 { 영업여부, 시작, 종료 }
- 키오스크 설정: 메시지, 복귀 시간, 비활성 타임아웃, 주문번호 접두사, 일일 리셋 여부

**인터랙션:**
| 동작 | 결과 |
|------|------|
| "저장" 클릭 | 유효성 검증 → 성공 시 저장 + 성공 토스트 |
| 영업시간 "전체 동일 적용" | 월요일 값을 화~일에 복사 |
| 유효성 실패 | 인라인 에러 메시지 |
| 시작 시간 > 종료 시간 | "종료 시간은 시작 시간 이후여야 합니다." 에러 |
