# 점심 모집 앱 - 화면 설계

> 모바일 웹 우선 설계. 데스크톱은 반응형으로 지원.
> 모바일 기준 폭: 375px~430px. 데스크톱: max-width 640px 중앙 정렬 (앱 느낌).

---

## 1. 랜딩 페이지

**경로**: `/`

**목적**: 서비스 소개, 워크스페이스 생성 또는 참여 유도

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | 앱 로고/이름 |
| 히어로 | 한 줄 카피 ("오늘 점심, 같이 먹을 사람?"), 간단한 일러스트 또는 스크린샷 |
| CTA 버튼 | "워크스페이스 만들기" (primary), "초대 링크로 참여" (secondary) |
| 간단 설명 | 3단계 사용법 (만들기 -> 초대 -> 모집) |

### 데이터
- 없음 (정적 페이지)

### 인터랙션
- "워크스페이스 만들기" 클릭 -> `/create` 이동
- "초대 링크로 참여" 클릭 -> 초대 링크 입력 모달 또는 안내 텍스트

### 반응형
- 모바일: 세로 스택 레이아웃
- 데스크톱: 동일 (중앙 정렬, max-width 640px)

---

## 2. 워크스페이스 생성 페이지

**경로**: `/create`

**목적**: 새 워크스페이스 생성

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | 뒤로가기, "워크스페이스 만들기" 타이틀 |
| 폼 | 워크스페이스 이름 (필수), 설명 (선택), 내 닉네임 (필수) |
| 버튼 | "만들기" (primary, 폼 유효 시 활성화) |

### 데이터
- 입력: 워크스페이스 이름, 설명, 닉네임
- 출력 (생성 후): 초대 링크, 관리 링크

### 인터랙션
- 필수 필드 미입력 시 버튼 비활성화
- "만들기" 클릭 -> API 호출 -> 성공 시 완료 화면 (모달 또는 새 페이지)
- 완료 화면에서 "초대 링크 복사", "시작하기" 버튼

### 필드 상세

| 필드 | 타입 | 필수 | 제약 | 플레이스홀더 |
|------|------|------|------|-------------|
| 워크스페이스 이름 | text | O | 2~30자 | "예: ABC테크 개발팀" |
| 설명 | textarea | X | 100자 이내 | "예: 개발팀 점심 모임 공간" |
| 내 닉네임 | text | O | 2~10자 | "예: 홍길동" |

---

## 3. 초대 참여 페이지

**경로**: `/:workspaceSlug/invite/:inviteCode`

**목적**: 초대 링크를 통해 워크스페이스에 참여

### 구성요소

| 영역 | 내용 |
|------|------|
| 로고 | 앱 로고 |
| 안내 문구 | "**{워크스페이스 이름}**에 참여합니다" |
| 폼 | 닉네임 입력 (필수) |
| 버튼 | "참여하기" (primary) |

### 데이터
- URL 파라미터에서 워크스페이스 정보 조회
- 입력: 닉네임

### 인터랙션
- 유효하지 않은 초대 코드: 에러 화면 ("유효하지 않은 링크입니다")
- 이미 참여한 워크스페이스 (쿠키 확인): 바로 메인으로 리다이렉트
- "참여하기" 클릭 -> API 호출 -> 성공 시 메인 화면으로 이동

---

## 4. 워크스페이스 선택 페이지

**경로**: `/workspaces`

**목적**: 여러 워크스페이스에 소속된 경우 선택

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | "내 워크스페이스" 타이틀 |
| 리스트 | 워크스페이스 카드 목록 (이름, 멤버 수, 마지막 접속) |
| 하단 | "새 워크스페이스 만들기" 링크 |

### 데이터
- 쿠키에 저장된 워크스페이스 토큰 목록 -> 각 워크스페이스 정보 조회

### 인터랙션
- 워크스페이스 카드 클릭 -> 해당 워크스페이스 메인으로 이동
- "새 워크스페이스 만들기" -> `/create`로 이동

### 표시 조건
- 워크스페이스가 1개: 이 화면 건너뛰고 바로 메인으로 이동
- 워크스페이스가 0개: 랜딩 페이지로 리다이렉트

## 5. 메인 화면 (달력 + 오늘 모집글)

**경로**: `/:workspaceSlug`

**목적**: 핵심 화면. 달력으로 날짜별 모집글 확인, 오늘/선택한 날짜의 모집글 목록 표시

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | 워크스페이스 이름, 설정 아이콘 (우측), 프로필 아이콘 (좌측) |
| 달력 | 월간 미니 달력. 모집글 있는 날짜에 점(dot) 표시 |
| 날짜 라벨 | 선택된 날짜 표시 ("2월 24일 월요일", "오늘") |
| 모집글 목록 | 선택된 날짜의 모집글 카드 리스트 |
| FAB | "모집하기" 플로팅 액션 버튼 (우하단) |

### 달력 상세

- 월간 뷰 (고정, 접기/펼치기 없음. 모바일에서 컴팩트하게 표시)
- 이전/다음 달 화살표
- 오늘 날짜: primary 색상 원형 배경
- 선택된 날짜: 테두리 강조
- 모집글 있는 날짜: 날짜 아래에 작은 점(dot)
  - 모집 중인 글이 있으면: primary 색상 점
  - 전부 마감이면: neutral 색상 점

### 모집글 카드 구성

```
+------------------------------------------+
|  짬뽕                           12:00    |
|  만리장성 | 김개발                        |
|  [참여 2/4]                    [모집중]   |
+------------------------------------------+
```

| 요소 | 설명 |
|------|------|
| 메뉴명 | 볼드, 큰 글씨 |
| 시간 | 우측 상단 |
| 장소 | 회색 텍스트 (없으면 미표시) |
| 모집자 | 회색 텍스트 |
| 참여 현황 | "2/4" 형태 |
| 상태 뱃지 | 모집중 (primary) / 마감 (neutral) / 인원마감 (warning) |

### 데이터
- 월간 달력: 해당 월의 날짜별 모집글 유무/개수 (가벼운 API)
- 모집글 목록: 선택된 날짜의 모집글 리스트 (메뉴, 시간, 장소, 모집자, 참여현황, 상태)

### 인터랙션
- 달력 날짜 클릭 -> 해당 날짜의 모집글 목록 로드
- 모집글 카드 클릭 -> 모집글 상세 화면으로 이동
- FAB "모집하기" 클릭 -> 모집글 작성 화면 (선택된 날짜 기본값)
- 설정 아이콘 -> 워크스페이스 설정
- 프로필 아이콘 -> 프로필 설정

### 빈 상태
- 선택된 날짜에 모집글이 없을 때: "아직 모집글이 없습니다" + "첫 모집글을 올려보세요!" 버튼

### 반응형
- 모바일: 달력 (상단) + 모집글 목록 (하단) 세로 스택
- 데스크톱: 동일 레이아웃 (max-width 640px)

## 6. 모집글 상세 화면

**경로**: `/:workspaceSlug/gatherings/:gatheringId`

**목적**: 모집글의 전체 정보와 참여자 확인, 참여/취소 동작

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | 뒤로가기, 더보기 메뉴 (수정/삭제, 모집자만) |
| 상태 뱃지 | 모집중 / 마감 / 인원마감 |
| 메뉴명 | 큰 타이틀 |
| 정보 블록 | 아이콘 + 텍스트 리스트: 장소, 날짜, 시간, 인원 (2/4명) |
| 메모 | 모집자가 작성한 메모 (있을 경우) |
| 모집자 | "김개발 님이 모집" |
| 참여자 목록 | 참여자 닉네임 리스트 (아바타 + 이름). 모집자에 왕관 아이콘 |
| 액션 버튼 | 하단 고정. "참여하기" 또는 "참여 취소" (풀 너비) |

### 모집자 전용 요소

| 요소 | 설명 |
|------|------|
| 더보기 메뉴 | 수정, 삭제, 마감하기 |
| 마감 버튼 | "마감하기" (모집 중일 때만 표시) |

### 데이터
- 모집글 상세: 메뉴, 장소, 날짜, 시간, 최대인원, 메모, 상태, 모집자
- 참여자 목록: 닉네임, 참여 시간
- 내 참여 여부: 참여 중이면 "참여 취소" 버튼, 아니면 "참여하기" 버튼

### 인터랙션
- "참여하기" 클릭 -> API 호출 -> 참여자 목록 업데이트, 버튼 "참여 취소"로 전환
- "참여 취소" 클릭 -> 확인 다이얼로그 -> API 호출 -> 업데이트
- "수정" (모집자) -> 수정 폼 화면으로 이동
- "삭제" (모집자) -> 확인 다이얼로그 -> 삭제 후 메인으로 이동
- "마감하기" (모집자) -> 확인 다이얼로그 -> 마감 처리

### 마감 상태일 때
- 참여/취소 버튼 비활성화 또는 숨김
- "이 모집은 마감되었습니다" 안내
- 참여자 목록은 그대로 표시

## 7. 모집글 작성/수정 화면

**경로**: `/:workspaceSlug/gatherings/new` (작성) / `/:workspaceSlug/gatherings/:gatheringId/edit` (수정)

**목적**: 모집글 작성 또는 수정

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | 뒤로가기 (X), "모집글 작성" / "모집글 수정" 타이틀 |
| 폼 | 아래 필드 상세 참고 |
| 버튼 | "올리기" / "수정하기" (하단 고정, primary, 풀 너비) |

### 필드 상세

| 필드 | 타입 | 필수 | 기본값 | UI | 플레이스홀더/설명 |
|------|------|------|--------|-----|-----------------|
| 메뉴명 | text | O | - | Input | "예: 짬뽕, 돈까스" |
| 식당/장소 | text | X | - | Input | "예: 회사 앞 만리장성" |
| 날짜 | date | O | 오늘 (또는 달력에서 선택한 날짜) | 달력 선택 | - |
| 시간 | time | O | 12:00 | 시간 선택 (30분 단위 드롭다운) | 11:00 ~ 14:00 범위 |
| 최대인원 | number | O | 4 | 스테퍼 (+/-) 또는 드롭다운 | 2~10명 |
| 메모 | textarea | X | - | Textarea | "예: 탕수육 시켜서 나눠먹어요" |

### 데이터
- 작성: 초기값 (날짜, 시간, 인원 기본값)
- 수정: 기존 모집글 데이터 로드

### 인터랙션
- 필수 필드 미입력 시 "올리기" 버튼 비활성화
- "올리기" 클릭 -> API 호출 -> 성공 시 모집글 상세 화면으로 이동
- "X" (닫기) 클릭 -> 내용이 있으면 "작성을 취소하시겠습니까?" 확인 -> 메인으로 이동
- 과거 날짜 선택 불가

### 시간 선택 옵션
- 11:00, 11:30, 12:00, 12:30, 13:00, 13:30, 14:00
- 오늘 날짜이고 이미 지난 시간은 비활성화

## 8. 워크스페이스 설정 화면

**경로**: `/:workspaceSlug/settings`

**목적**: 워크스페이스 정보 확인, 초대 링크 관리, 멤버 조회

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | 뒤로가기, "설정" 타이틀 |
| 워크스페이스 정보 | 이름, 설명 (관리자: 수정 가능, 일반 멤버: 읽기 전용) |
| 초대 링크 | 링크 텍스트 + "복사" 버튼 |
| 멤버 목록 | 닉네임 리스트 (멤버 수 표시) |
| 하단 | "워크스페이스 나가기" 링크 (danger 색상) |

### 관리자 전용 (관리 링크로 접속한 경우)

| 요소 | 설명 |
|------|------|
| 이름/설명 수정 | 인라인 편집 또는 수정 모드 |
| 초대 링크 재발급 | "링크 재발급" 버튼 (기존 링크 무효화 경고) |

### 데이터
- 워크스페이스 정보: 이름, 설명, 생성일
- 초대 링크
- 멤버 목록: 닉네임, 참여일

### 인터랙션
- "복사" 클릭 -> 클립보드에 링크 복사, 토스트 "복사되었습니다"
- "링크 재발급" -> 확인 다이얼로그 "기존 링크가 무효화됩니다" -> 재발급
- "워크스페이스 나가기" -> 확인 다이얼로그 -> 쿠키에서 해당 워크스페이스 토큰 삭제

---

## 9. 프로필 설정 화면

**경로**: `/:workspaceSlug/profile`

**목적**: 닉네임 변경

### 구성요소

| 영역 | 내용 |
|------|------|
| 헤더 | 뒤로가기, "프로필" 타이틀 |
| 닉네임 | 현재 닉네임 표시, 수정 Input |
| 버튼 | "저장" (변경 사항이 있을 때만 활성화) |
| 워크스페이스 전환 | "다른 워크스페이스로 이동" 링크 (복수 소속 시) |

### 데이터
- 현재 닉네임

### 인터랙션
- 닉네임 변경 후 "저장" -> API 호출 -> 토스트 "닉네임이 변경되었습니다"
- "다른 워크스페이스" 클릭 -> `/workspaces`로 이동

---

## 화면 전체 라우팅 맵

```
/                                         랜딩 페이지
/create                                   워크스페이스 생성
/workspaces                               워크스페이스 선택 (복수 소속 시)
/:workspaceSlug/invite/:inviteCode        초대 참여
/:workspaceSlug                           메인 (달력 + 모집글 목록)
/:workspaceSlug/gatherings/new            모집글 작성
/:workspaceSlug/gatherings/:gatheringId   모집글 상세
/:workspaceSlug/gatherings/:gatheringId/edit   모집글 수정
/:workspaceSlug/settings                  워크스페이스 설정
/:workspaceSlug/profile                   프로필 설정
```

## 공통 UI 요소

### 헤더 (앱 바)
- 높이: 48px (모바일)
- 좌측: 뒤로가기 화살표 또는 프로필 아이콘
- 중앙: 페이지 타이틀
- 우측: 액션 아이콘 (설정, 더보기 등)

### 토스트 메시지
- 하단에서 올라오는 임시 알림
- 3초 후 자동 사라짐
- 종류: 성공 (초록), 에러 (빨강), 정보 (파랑)

### 확인 다이얼로그
- 중앙 모달
- 제목, 설명, 취소/확인 버튼
- 삭제 등 위험한 동작: 확인 버튼에 danger 스타일

### FAB (Floating Action Button)
- 우하단 고정
- "+" 또는 "모집하기" 텍스트
- primary 색상
- 메인 화면에서만 표시

### 로딩 상태
- 스켈레톤 UI (카드, 리스트)
- 버튼 클릭 시: 스피너 + 비활성화 (중복 클릭 방지)

### 빈 상태 (Empty State)
- 일러스트 + 안내 문구
- CTA 버튼 (해당되는 경우)
